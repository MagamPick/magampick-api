# Product

## Product Summary
베이커리·카페의 마감 재고를 주변 소비자가 저렴하게 픽업할 수 있도록 연결하는 로컬 마감 세일 픽업 플랫폼. 사장은 폐기 대신 수익 회수, 소비자는 할인 구매.

## Target Users
- Primary: 매장 인근(반경 1-3km) 거주/근무하는 소비자
- Primary: 베이커리·카페 점주 (사장)
- Secondary: 관리자 (운영자)
- Not targeting: 매장에서 거리가 먼 사용자, 배달을 원하는 사용자, 베이커리·카페 외 업종

## Core Value
1. 소비자: 주변 매장의 마감 할인 상품을 실시간으로 알 수 있다
2. 사장: 폐기될 재고를 수익으로 회수할 수 있다

## Core Flow
1. 사장: 마감 임박 상품 등록 (수량 / 가격 / 픽업 가능 시간대)
2. 소비자: 위치 기반으로 상품 탐색
3. **당일 결제 → 당일 픽업** (예약·익일 픽업 불가)
4. 매장 방문 픽업 (배달 없음)

## Platform

**클라이언트 구성**

| 클라이언트 | 종류 | 푸시 알림 | 위치 |
|---|---|---|---|
| 소비자 웹 | **PWA** | FCM ✅ | Web Geolocation ✅ |
| 사장 웹 | **PWA** | FCM ✅ | Web Geolocation (선택) |
| 관리자 웹 | 일반 Web | 미사용 | 미사용 |

- **PWA** = Progressive Web App (Service Worker + Manifest, 설치 가능)
- **푸시 알림**: **FCM (Firebase Cloud Messaging)** — PWA 환경에서 Web Push 메커니즘으로 수신. iOS Safari 16.4+ 부터 지원
- **위치 정보**: Web Geolocation API (HTTPS 필요)
- **네이티브 모바일 앱 미지원** (iOS / Android 앱스토어 출시 X)

## Order States
`주문접수 → 준비중 → 준비완료 → 픽업완료`

- **주문접수**: 결제 완료, 사장 승인 대기
- **준비중**: 사장이 주문 승인
- **준비완료**: 매장 픽업 가능
- **픽업완료**: 거래 완료

## Out of Scope

### 비즈니스
- 배달 — 픽업 전용
- 베이커리·카페 외 업종 — 한식/일식/중식 등 일반 음식점 미지원
- 식자재/원재료 거래 — 완성된 상품만
- B2B 대량 주문 — B2C 소량 픽업만

### 결제
- 현장 결제 — 선결제만
- 무통장 입금 — 카드/간편결제만
- 외화 결제 — 원화만

### 사용자
- 비회원 구매
- 해외 사용자 — 국내 한정

### 플랫폼
- **네이티브 모바일 앱** (iOS / Android) — PWA only
- **애플 로그인** — PWA 라 iOS 앱스토어 정책 무관

### 운영
- 24시간 영업 매장 — 영업시간/마감시간 기반 운영이라 부적합
- 익일 예약 픽업 — 당일 픽업만

## Business Constraints
- 토스페이 결제 시스템 사용 (PG사 계약)
- 카카오맵/주소 API 사용
- 국세청 API로 사업자 인증 (매장 등록 시 필수)
- Firebase Cloud Messaging (FCM) 사용 (푸시 알림)

→ 수수료·정산·환불·포인트·쿠폰·알림·노쇼 등 운영 정책: [`policy.md`](policy.md) 참조

## Decision Principles
- 실시간성 > 정밀도 (떨이 알림은 즉시 발송이 우선)
- 픽업 단순함 > 풍부한 기능
- 매장 신뢰성 > 빠른 등록 (사업자 인증 + 관리자 승인 필수)
- 위치 기반 우선 (모든 탐색의 기준은 위치)
